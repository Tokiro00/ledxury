PASO 1: MIGRACIONES DE BASE DE DATOS (EN ORDEN)
============================================================

Ejecutar estos SQL en la DB mamcontabilidad, EN ESTE ORDEN:

  1) db/migrations/018_client_tokens.sql
     - Tabla: client_tokens
     - Para: tokens de acceso directo de clientes (PWA sin login)

  2) db/migrations/019_notifications.sql
     - Tabla: notifications
     - Para: notificaciones in-app cuando cliente hace pedido

  3) db/migrations/020_chat_messages.sql
     - Tabla: chat_messages
     - Para: mensajeria bidireccional cliente <-> vendedor

  4) db/migrations/021_catalog_overrides.sql
     - Tabla: catalog_overrides
     - Para: control manual de ofertas/remate desde el ERP

IMPORTANTE: Son CREATE TABLE IF NOT EXISTS, no rompen nada
si ya existen. Pero el ORDEN importa.

PASO 3: CONFIGURACION
============================================================

En application/config/config.php o secrets.php agregar
(si no existe ya):

  $config['anthropic_api_key'] = 'sk-ant-...';

Esto activa el asistente IA en la PWA de clientes.
Sin esta key, el chat de IA no funciona pero todo lo demas si.

============================================================
PASO 4: VERIFICAR
============================================================

Probar estas URLs despues de montar:

  PWA Vendedores:  {base_url}/pwa/
  PWA Clientes:    {base_url}/pwa/clientes/?t=TOKEN_DE_PRUEBA
  PWA Ejecutiva:   {base_url}/pwa/exec/

  API V1:          POST {base_url}/api/v1/login
  API Client:      GET  {base_url}/api/client/validate?t=TOKEN
  API Executive:   GET  {base_url}/api/exec/dashboard

Para generar un token de prueba para la PWA de clientes:
  - Entrar a la PWA de vendedores
  - Ir a un cliente > "Enviar catalogo"
  - Eso genera un registro en client_tokens y da el link

NOTAS
============================================================

- La PWA de clientes funciona SIN la API key de Anthropic,
  solo el boton de chat IA queda deshabilitado.
- Los descuentos del catalogo se calculan automaticamente
  basados en dias sin venta y stock.
- catalog_overrides permite control manual desde el ERP
  (excluir productos, fijar precios, forzar tabs).
- El JWT sigue igual (HS256, 7 dias, secret en secrets.php).
- NO se modifico nada de contabilidad ni inventario.
